# TicTacToe-Pro-Project
Проєкт представляє собою просунуту десктопну гру, що реалізує турнірний режим із динамічною зміною складності та розміру поля. Додаток побудований на базі C# WinForms із застосуванням сучасних паттернів проєктування та принципів чистого коду.

Основні можливості
Турнірний режим: Автоматичне проведення 3 раундів із поступовим ускладненням.

Динамічне поле: Масштабування ігрової сітки від 3x3 до 5x5 залежно від раунду.

Інтелектуальний ШІ: Використання алгоритму Minimax для непереможної гри та EasyStrategy для швидких партій.

Система Undo/Redo: Можливість скасування та повторення ходів завдяки збереженню історії станів.

Аналітика та Досягнення: Відстеження швидкості перемоги, контролю кутів поля та збір загальної статистики сесії.

Таблиця лідерів: Збереження результатів гравців у файл та їх відображення в інтерактивному рейтингу.

Глобальний аудит: Система логування подій (FileLogger) для відстеження ігрового процесу та помилок.

Принципи та Патерни
Programming Principles
SRP (Single Responsibility): Розділення логіки на GameEngine (стан поля), TournamentManager (раунди) та ScoreService (рекорди).

DIP (Dependency Inversion): Форма залежить від інтерфейсів IGameLogic та IMoveStrategy, а не від конкретних реалізацій.

DRY (Don't Repeat Yourself): Універсальний алгоритм перевірки переможця CheckLine працює для будь-якого розміру поля.

Separation of Concerns: Чітке розділення на папки Core, Models, Services та Views.

Design Patterns
Strategy Pattern: Реалізовано в алгоритмах ходів бота (IMoveStrategy). Це дозволяє гнучко змінювати рівень складності (Easy, Minimax).

Factory Method: Клас StrategyFactory централізовано створює потрібну стратегію на основі номеру раунду.

Command Pattern: Клас CommandHistory зберігає знімки поля MoveSnapshot для реалізації функцій скасування ходу.

Singleton: Застосовано для ScoreService та AnalyticsService, забезпечуючи єдиний доступ до даних у всій системі.

Техніки рефакторингу
Extract Method: Складні блоки коду в Form1 (напр. генерація кнопок) винесені в окремі методи.

Rename Variable: Технічні назви замінені на змістовні (botButton, winnerSymbol).

Move Method: Логіку оцінки поля перенесено в спеціалізований клас BoardEvaluator.

Replace Magic Number with Symbolic Constant: Використання констант для розмірів поля та балів.

Як запустити локально

Відкрийте проєкт:
Запустіть файл TicTacToe.UI.sln через Visual Studio 2022.

Запустіть застосунок:
Натисніть F5 або кнопку Start.

Тестові дані:
Таблиця лідерів завантажиться автоматично з файлу при першому запуску гри.
